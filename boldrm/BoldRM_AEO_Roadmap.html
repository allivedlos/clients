What is AEO?

Answer Engine Optimization

AEO is the practice of making your business the answer that AI-powered tools deliver — not just a result to click. When someone asks "What's the best home staging company in San Francisco?" the goal is for boldRM to be the business ChatGPT and Perplexity name in their response.

The book distinguishes two types of AI engines: Pre-trained (Claude, Llama — frozen knowledge) and Real-time (Perplexity, ChatGPT with browsing, Google AI Mode — actively crawl the web). Real-time engines are the primary opportunity.

Key research finding Of 7,923 URLs analyzed — 57.8% came from URLs ranking on Google/Bing, but 42.2% came from sources not ranking on traditional search at all. AI doesn't just scrape the top 3 results. Brand websites and blogs are the #1 and #2 most-cited source types across all three engines.

The Framework

The BRAIN Framework

Every phase of this roadmap maps to one of the five BRAIN pillars defined in The AEO Blueprint. Together they form the strategy for becoming the AI-recommended answer in your market.

B
Brand Representation
Consistent NAP + brand presence across all platforms AI trusts. Inconsistency = AI won't cite you. Even punctuation matters — "650-988-5840" and "650.988.5840" are different strings to an LLM.
R
Recognition / Authority
Deep, original, experience-based content only boldRM could produce. No AI-recycled articles — that's the "death spiral." Unlinked brand mentions now count as authority signals.
A
Authority & Citations
Be cited on external platforms — agent blogs, Inman, RESA, Houzz, Reddit. AI triangulates trust from Social Proof + Structured Fact + First-Party Truth (your site).
I
Indexability
JSON-LD schema (server-side only, never via GTM), llms.txt, clean URLs, sitemap, entity linking. AI bots can't execute JavaScript — schema injected via GTM is completely invisible to them.
N
Net New Content
Every piece must add genuinely new information to the web. AI writes articles → AI cites those → content gene pool shrinks → LLMs stop treating it as authoritative. This is the death spiral.
Platform Strategy

AI Engine "Personalities"

Each AI engine has a distinct citation strategy. A one-size-fits-all approach won't maximize visibility. Understanding these differences shapes how we prioritize each phase.

EngineCitation StrategyWhat This Means for boldRM
Perplexity Strongly prefers page-level authority; 79% of citations rank on Google/Bing. Favors official brand websites above all else. City pages + LocalBusiness schema must be strong. Perplexity goes straight to your site first.
ChatGPT Balanced approach: high-traffic domains (avg 9.9M) + page relevance. 56% of citations rank on traditional search. Domain authority + specific FAQ pages matter equally. Build both simultaneously.
Google AI Mode Paradoxically ignores its own search rankings — 63% of citations come from pages NOT on Google/Bing. Favors blog/expert commentary. Third-party posts about boldRM, Inman mentions, and agent blogs citing you are critical here specifically.
Phase 1 Fix & Index — Critical Foundation Week 1–2 · Do these first

The book is clear: schema markup is "the single most important and unambiguous factor" for AI visibility. But there's a critical trap — AI bots cannot execute JavaScript. If schema is injected via GTM or client-side JS, AI crawlers are completely blind to it. Fix these issues before any content or authority work.

Critical — JavaScript schema is invisible to AI crawlers Platforms like GPTBot, ClaudeBot, and PerplexityBot do not execute JavaScript. Any JSON-LD added after the page loads via GTM is invisible. Webflow renders server-side by default — this is an advantage. Add all schema directly in Webflow's Page Settings → Custom Code.

Update robots.txt to explicitly allow AI bots

Allow GPTBot, PerplexityBot, ClaudeBot, OAI-SearchBot, and Googlebot explicitly. Add a pointer to the sitemap. The book recommends this as a foundational indexability step — don't accidentally block AI crawlers.

Core technical
Add FAQPage JSON-LD schema to all city pages

The book's #1 technical mandate. FAQPage schema lets AI engines extract Q&A pairs as structured data. Without it, LLMs can't reliably cite your FAQ answers. Must be embedded directly in the raw HTML — not via GTM. Each city page has 20+ Q&As that need to be encoded in JSON-LD and added via Webflow's Page Settings → Custom Code → Before </body>.

→Webflow → Each city page → Page Settings → Custom Code (server-side only)
📖 AEO Blueprint Ch.10: "Schema as the Target: AI crawlers specifically seek valid and comprehensive Schema Markup (FAQPage, Product, Review, etc.)."
Add LocalBusiness + sameAs schema to homepage

The book specifically calls out sameAs schema as a way to "link your brand to the web's most authoritative sources — Wikidata, Crunchbase — that AI relies on to build its semantic relationships." Include: business name, address, phone, URL, service area, aggregate rating, and sameAs links to Yelp, Google Business, LinkedIn, Facebook, and Wikidata.

→Webflow → Homepage → Page Settings → Custom Code
📖 AEO Blueprint Ch.10: "sameAs helps search engines connect the dots about your brand and lights up your visibility in AI-powered search results."
Create llms.txt file at root domain

The book dedicates an entire section to this. llms.txt is a Markdown-formatted file at boldrm.com/llms.txt that gives AI crawlers a structured map of your site. Reduces the token cost for crawlers — LLMs skip sites that are expensive to read. Include: H1 site name, summary block, links to key pages (city pages, services, blog), contact info. Also create llms-full.txt bundling all documentation.

→Webflow Hosting → Custom file upload or redirect to a hosted Markdown file
📖 AEO Blueprint Ch.9: "llms.txt is like a backstage pass for AI bots — giving them all the metadata and context they need to truly comprehend what your site is all about."
Verify all city pages are indexed in Google Search Console

Confirm every /cities/ page appears in GSC and request indexing for any that aren't. The book notes that 57.8% of AI-cited URLs rank on traditional search — your city pages must be indexed to be in consideration for Perplexity and ChatGPT citations.

→Google Search Console → URL Inspection → Request Indexing

Phase 3 Deep Authority — Original Content + Topical Depth Week 4–8 · The BRAIN "R" + "A" layer

The book introduces Topical Authority — AI rewards brands that go deep, not wide. Don't try to be a generalist; own the "home staging in the Bay Area" niche completely. Pages cited by AI average 2,288–3,400 words depending on the engine. But the #1 rule: every article must be net-new.

The AI death spiral — avoid this Using ChatGPT to write staging articles → those get indexed → AI engines cite them when writing new articles → content gene pool shrinks → LLMs stop treating it as authoritative. Every boldRM article must use real data: actual addresses, real sale prices, genuine before/after outcomes.
Original content — real data only

Topics: (1) What we learned staging 50 SF condos — with real stats; (2) Victorian vs Eichler staging differences; (3) The $X stage that got $Y over asking in [city] — real address, real outcome; (4) ROI of staging by Bay Area city with actual sale data. Aim for 2,000–3,400 words per article. Real numbers only.

→Webflow CMS Blog — publish minimum one per week
📖 AEO Blueprint Ch.9: "Produce original, niche content: The more specific and specialized your content, the more AI will recognize your expertise."

Add Article/BlogPosting schema to all blog posts

Add @id property and entity linking to schema markup

The book dedicates a full section to the @id property: "It gives each entity a globally unique identifier — like a fingerprint for your content." Add @id to the Organization schema on the homepage (e.g., "https://www.boldrm.com/#organization") and reference this @id in Article schema author fields. This tells AI that all content links back to one authoritative source entity.

→Webflow → Homepage schema + Blog template schema — update existing JSON-LD
📖 AEO Blueprint Ch.10: "The @id property is a gateway to the larger web of knowledge, linking content to trusted external sources like Wikidata."
Add Review + AggregateRating schema using existing Yelp reviews

The homepage already displays 3 Yelp reviews with names, photos, and locations. Review sites are the #3 most-cited source type across all AI engines — they layer in "social proof and consumer sentiment." Mark up these reviews with Review schema and an AggregateRating block. This enables star ratings in Google results and makes reviews available as structured data for AI to cite.

→Webflow Homepage → Custom Code → add Review + AggregateRating JSON-LD
Build content knowledge graph via internal linking strategy

The book instructs: "Build your own Content Knowledge Graph — linking your content both internally and externally so AI can understand the relationships between things on your site." Create a linking map: every city page links to related blog posts + the services page + the cost calculator. Every blog post links to relevant city pages. This creates the semantic network AI uses to determine topical authority.

→Webflow CMS → add cross-links between city pages, blog posts, and services
📖 AEO Blueprint Ch.9: "Linking your content both internally and externally makes it stand out as reliable and interconnected, improving entity recognition."
Publish original content on LinkedIn Pulse

Publish 2–3 articles that are unique to the platform (not duplicates of the blog): "What Bay Area Agents Need to Know About Staging ROI in 2026" / "5 Mistakes That Cost Sellers $20K at Open House." Each article builds a citation signal on a domain AI trusts.

→LinkedIn → Write Article → publish with link back to relevant city pages

Phase 5 Automate & Monitor — The Compounding System Week 8–12 · The system that grows itself

This phase turns AEO from a one-time project into a self-compounding system. Every staging project automatically generates a new, original, indexed case study. The book recommends content repurposing with AI tools and automation as the sustainable way to maintain freshness signals without burning out.

Multimodal content strategy The book stresses showing up in multiple formats — text, video, audio. 67% of Gen Z starts searches on Instagram, 64% use TikTok instead of Google. The photo-to-CMS automation below aligns with this: every stage becomes a story published across Webflow blog, LinkedIn Pulse, and social, maximizing citation surface area.
Build photo → Claude → Webflow CMS automation

Full automation flow: Photographer uploads to Dropbox → Zapier trigger → property details logged in Airtable → Claude API writes a unique case study (story + FAQs + social caption + meta description + JSON-LD schema block) → Zapier auto-drafts in Webflow CMS → Admin reviews → 1-click publish → Zapier notifies agent subscriber list via MailerLite. Every stage becomes one new indexed, schema-marked article with net-new proprietary data.

→Zapier + Claude API + Airtable + Webflow CMS + Dropbox + MailerLite
📖 AEO Blueprint Ch.9: "Repurposing content builds serious authority. You start showing up everywhere, saying the same smart stuff in different ways."
Set up weekly AI citation monitoring

Every week: query ChatGPT (with browsing), Perplexity, and Google AI Mode with target queries: "best home staging company in San Francisco / Burlingame / Los Altos / Millbrae / Mountain View," "how much does home staging cost in the Bay Area," "who does home staging in [city]." Log results in Airtable. Track citation frequency week over week — this is the primary KPI of the entire program.

→Manual queries (20 min/week) → logged in Airtable citation tracker
Build AEO monitoring dashboard in Airtable

Track weekly: GSC query impressions by city, AI citation check results per engine, new backlinks acquired, city page rankings, blog post indexed status, schema validation status (Google's Rich Results Test). The book recommends monitoring branded search volume as a key indicator — "if you work on AI visibility, your branded searches will start to rise." Populate via Zapier from GSC exports.

→Airtable dashboard + Zapier automation from Google Search Console
Validate all schema monthly with Google's Rich Results Test

The book's research found that URLs with valid schema have 65% higher average page traffic than URLs with failed/no schema (~19,000 vs ~11,500 avg visits). Run monthly validation on all city pages, homepage, and recent blog posts. Check Google Search Console for "Structured Data Errors" report. Errors in schema can confuse AI systems — ensure data matches what's visible to users.

→Google Rich Results Test + Google Search Console → Enhancements tab
📖 AEO Blueprint Research: "URLs with 'Passed' schema validation have noticeably higher average traffic (~19,000) than URLs with 'Failed/No Schema' (~11,500)."
